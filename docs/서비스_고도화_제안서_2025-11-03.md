#### 5. **게이미피케이션**

## 교사 니즈 분석

### 💔 교사 고충 Top 10

#### 시간 관리 문제
1. **퀴즈 제작 시간 과다** (평균 40-60분/퀴즈)
   - 수업 준비, 채점, 교실 관리 병행
   - 온라인 수업에서 더욱 심화

2. **수동 채점 업무 부담** (교사 시간의 30-50%)
   - 대규모 학급에서 필기 답안 검토 부담
   - 고차원 사고 평가 대신 맞춤법 검사에 시간 소비

3. **반복 행정 업무**
   - 다른 학급에 동일 유형 문제 재생성
   - 플랫폼별 형식 재조정

#### 품질 및 교육학적 과제
4. **고차원 사고력 문항 제작 어려움** (HOTS)
   - 단순 암기를 넘어선 문제 생성 난이도
   - Bloom's Taxonomy 정렬 필요

5. **수준별 차별화 어려움**
   - 다양한 학습자를 위한 난이도별 문항 제작
   - 대규모 개인화 피드백

6. **형성평가 도구 부족**
   - 수업 중 실시간 이해도 체크 필요
   - 개별 학생 진도 모니터링 어려움

#### 기술 및 플랫폼 한계
7. **Google Forms 제약**
   - 제한적 템플릿 (커스터마이징 시간 소요)
   - 전문 작업용 고급 기능 부족
   - 온라인 토론 포럼에서 사용 불가

8. **플랫폼 전환 피로감**
   - 시스템 간 수동 콘텐츠 이전
   - 플랫폼별 학생 데이터 불일치

#### 피드백 및 참여도 문제
9. **지연된 피드백 루프**
   - 학생이 결과를 며칠 대기
   - 즉각 교정 기회 상실

10. **전통적 퀴즈에 대한 학생 무관심**
    - 인터랙티브 및 게임화 요소 필요
    - 온라인/하이브리드 환경에서 주의력 유지 어려움

#### 2. **HOTS (고차원 사고력) 문항 생성** ⭐⭐⭐
**문제점:**
- 단순 암기 문제만 생성되는 경향
- Bloom's Taxonomy 정렬 어려움

**솔루션:**
```typescript
// Bloom's Taxonomy 수준별 프롬프트 엔지니어링
enum BloomLevel {
  Remember = "기억하기",      // 1단계: 사실 회상
  Understand = "이해하기",    // 2단계: 개념 설명
  Apply = "적용하기",         // 3단계: 상황 적용
  Analyze = "분석하기",       // 4단계: 요소 분해
  Evaluate = "평가하기",      // 5단계: 판단/비평
  Create = "창조하기"         // 6단계: 새로운 창출
}

interface HOTSQuizRequest {
  content: string;
  bloomLevel: BloomLevel;
  questionCount: number;
  grade: string;
}

// Gemini 프롬프트 강화:
// "분석하기(4단계) 수준의 문제를 생성하세요.
//  - 원인과 결과 관계를 파악하는 문제
//  - 증거를 평가하는 문제
//  - 패턴을 인식하는 문제"
```

**기대 효과:**
- 사고력 중심 평가
- 교육과정 성취기준 정렬
- 학생 고차원 사고력 발달

**구현 방법:**
1. `src/app/api/generate/route.ts`에 Bloom 수준 파라미터 추가
2. 수준별 프롬프트 템플릿 작성
3. UI에 Bloom 수준 선택 옵션 추가

**예상 개발 기간:** 2-3주

```typescript
// 기능:
```

**기대 효과:**

### Phase 2: 플랫폼 확장 (6개월)

#### 4. **다중 입력 형식 지원** ⭐⭐
**현재 상태:** 텍스트만 지원
**목표:** PDF, PPT, YouTube 영상, 이미지

**솔루션:**
```typescript
interface MultiFormatInput {
  type: 'text' | 'pdf' | 'pptx' | 'youtube' | 'image';
  content: File | string;
}

// PDF 처리: pdf-parse 라이브러리
```

#### 1. **즉시 구현 (2025년 11월-12월)**

**방법 A: forms.setPublishSettings() 메서드 추가**

```typescript
// src/lib/google-forms-api.ts (새 파일 생성)

import { google } from 'googleapis';

/**
 * Google Forms API를 사용하여 폼을 게시 상태로 설정
 *
 * @param formId - 게시할 폼의 ID
 * @param accessToken - Google OAuth 액세스 토큰
 */
export async function publishForm(
  formId: string,
  accessToken: string
): Promise<void> {
  const forms = google.forms({ version: 'v1' });

  try {
    await forms.forms.setPublishSettings({
      formId: formId,
      requestBody: {
        publishSettings: {
          // 폼을 게시 상태로 설정
          // published: true는 기본값이므로 생략 가능
        }
      },
      auth: accessToken
    });

    console.log(`✅ Form ${formId} successfully published`);
  } catch (error) {
    console.error(`❌ Failed to publish form ${formId}:`, error);
    throw new Error(`Form publishing failed: ${error.message}`);
  }
}

/**
 * 폼 생성 및 자동 게시 (2026년 3월 31일 이후 대비)
 *
 * @param formSpec - FormSpec 객체
 * @param accessToken - Google OAuth 액세스 토큰
 * @returns 생성된 폼 ID
 */
export async function createAndPublishForm(
  formSpec: any,
  accessToken: string
): Promise<string> {
  // 1. 폼 생성 (기존 로직)
  const forms = google.forms({ version: 'v1' });
  const createResponse = await forms.forms.create({
    requestBody: formSpec,
    auth: accessToken
  });

  const formId = createResponse.data.formId!;

  // 2. 폼 게시 (2026년 3월 31일 이후 필수)
  await publishForm(formId, accessToken);

  return formId;
}
```

#### 3. **권한 관리 추가 고려**

**선택적 기능: 특정 사용자와 폼 공유**

```typescript
/**
 * Drive API를 사용하여 폼을 특정 사용자와 공유
 *
 * @param formId - 공유할 폼의 ID
 * @param emails - 공유 대상 이메일 목록
 * @param role - 권한 ('reader', 'writer', 'commenter')
 * @param accessToken - Google OAuth 액세스 토큰
 */
export async function shareForm(
  formId: string,
  emails: string[],
  role: 'reader' | 'writer' | 'commenter',
  accessToken: string
): Promise<void> {
  const drive = google.drive({ version: 'v3' });

  for (const email of emails) {
    try {
      await drive.permissions.create({
        fileId: formId,
        requestBody: {
          type: 'user',
          role: role,
          emailAddress: email
        },
        auth: accessToken
      });

      console.log(`✅ Form ${formId} shared with ${email}`);
    } catch (error) {
      console.error(`❌ Failed to share form ${formId} with ${email}:`, error);
    }
  }
}
```

## 부록

### A. Google Forms API 변경 상세 가이드

#### 새로운 API 메서드: `forms.setPublishSettings()`

**요청 형식:**
```json
POST https://forms.googleapis.com/v1/forms/{formId}:setPublishSettings

{
  "publishSettings": {
    // 빈 객체로도 게시 가능 (기본값: published)
  }
}
```

**응답 형식:**
```json
{
  "formId": "1a2b3c4d5e6f",
  "publishSettings": {
    "published": true
  }
}
```

**에러 처리:**
```typescript
try {
  await publishForm(formId, accessToken);
} catch (error) {
  if (error.code === 403) {
    // 권한 없음: OAuth 스코프 확인
    console.error("Insufficient permissions to publish form");
  } else if (error.code === 404) {
    // 폼을 찾을 수 없음
    console.error("Form not found");
  } else {
    // 기타 에러
    console.error("Unknown error:", error);
  }

  // 사용자에게 에러 메시지 표시
  throw new Error("Failed to publish form. Please try again.");
}
```
